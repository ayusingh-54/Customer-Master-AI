# Deploying Customer Master AI on Render

Production-grade deployment guide for the testing/v1 release.

---

## Architecture

```
Render Web Service (Docker, starter plan $7/mo)
├── FastAPI REST API   — /api/v1/*
├── MCP SSE Server     — /mcp/sse  (Claude Desktop connects here)
├── Swagger UI         — /docs
└── Health check       — /health

Persistent Disk (1 GB)
└── /data/demo.db      — SQLite database (survives deploys)
```

---

## Quick Deploy (5 minutes)

### Option A — Render Dashboard (easiest)

1. Push this repo to GitHub (public or private).
2. Go to [render.com](https://render.com) → **New** → **Blueprint**.
3. Connect your GitHub repo — Render reads `render.yaml` automatically.
4. Set the **secret** env vars in the dashboard (see step 3 below).
5. Click **Apply** → deploy starts.

### Option B — render CLI

```bash
render blueprint apply --file render.yaml
```

## Step-by-Step

### 1. Push to GitHub

```bash
git init
git add .
git commit -m "feat: production Render deployment"
git remote add origin https://github.com/your-org/customer-master-ai.git
git push -u origin main
```

### 2. Connect repo in Render

- **New** → **Blueprint** → select your repo → **Connect**
- Render detects `render.yaml` and creates the service + disk

### 3. Set secret environment variables

In the Render dashboard → your service → **Environment** tab, set:

| Variable | Value | Notes |
|---|---|---|
| `ANTHROPIC_API_KEY` | `sk-ant-...` | From console.anthropic.com |

The `API_SECRET_KEY` is auto-generated by Render (`generateValue: true`).
**Copy it from the Environment tab** — you'll need it to call the REST API.

### 4. First deploy

Render builds the Docker image and starts the service.
- Initial build: ~3–5 minutes
- Health check: `GET /health` must return HTTP 200

### 5. Verify

```bash
# Replace with your Render URL
BASE=https://customer-master-ai.onrender.com

# Health check (no auth needed)
curl $BASE/health

# Search parties (auth required)
curl -X POST $BASE/api/v1/parties/search \
  -H "X-API-Key: YOUR_API_SECRET_KEY" \
  -H "Content-Type: application/json" \
  -d '{"query": "Acme"}'

# Open Swagger UI
open $BASE/docs
```

## Connect Claude Desktop (MCP)

Add to `%APPDATA%\Claude\claude_desktop_config.json` (Windows) or
`~/Library/Application Support/Claude/claude_desktop_config.json` (Mac):

```json
{
  "mcpServers": {
    "customer-master-ai": {
      "url": "https://customer-master-ai.onrender.com/mcp/sse",
      "headers": {
        "X-API-Key": "YOUR_API_SECRET_KEY"
      }
    }
  }
}
```

Restart Claude Desktop. The 22 agent tools appear in every conversation.

## Local Development

```bash
# Install dependencies
pip install -r requirements.txt

# Copy and edit env
cp .env.example .env
# → set ANTHROPIC_API_KEY

# Start server (Windows — uses uvicorn directly, not gunicorn)
uvicorn api.main:app --host 0.0.0.0 --port 10000 --reload

# Open docs
start http://localhost:10000/docs
```

| `WORKERS` | `1` | Gunicorn workers |

## Upgrading

Render redeploys automatically on every push to the connected branch.
The `/data` persistent disk retains the database across deploys.

To wipe and reseed the database (e.g., after schema changes):
```bash
# In Render Shell (dashboard → your service → Shell tab)
rm /data/demo.db
# Next request to any endpoint triggers init_db() and reseeds
```
